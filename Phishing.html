Ameaças Cibernéticas

Ataques de Phishing

Descrição:

Um funcionário recebe um e-mail aparentemente legítimo de seu banco, solicitando que ele clique em um link para verificar informações da conta. O link leva a uma página falsa que coleta suas credenciais.

Este site demonstra como os ataques de phishing funcionam e como identificar sinais de um e-mail falso.

Como o phishing funciona:

  • O golpista envia um e-mail, mensagem de texto ou ligação que parece legítima.
  • Pode se passar por bancos, empresas conhecidas (Netflix, Amazon) ou colegas de trabalho.
  • Cria senso de urgência ou medo com mensagens como:
    • "Sua conta será bloqueada!"
    • "Detectamos uma tentativa de invasão."
    • "Clique aqui para evitar problemas."
  • O e-mail contém um link que parece verdadeiro, mas direciona para uma página falsa (clonada).
  • Pode vir com um anexo infectado que instala vírus no dispositivo.
  • A vítima insere login, senha, CPF ou dados bancários na página falsa.

Como se proteger:

  • Verifique o remetente do e-mail. Desconfie de endereços estranhos.
  • Não clique em links ou baixe anexos de fontes desconhecidas.
  • Verifique a URL do site. Sites legítimos usam HTTPS.
  • Desconfie de mensagens que criam urgência ou medo.
  • Use autenticação em duas etapas sempre que possível.
  • Mantenha seu software antivírus atualizado.

Soluções:

  • Auxiliar os usuários a utilizarem autenticação de dois fatores.
  • Utilizar um gerenciador de senhas.
  • Usar e-mails temporários para cadastros em sites desconhecidos.
  • Utilizar um firewall ou antivírus confiável.

Ataques de Ransomware

Descrição:

Um usuário baixa um arquivo de um site desconhecido. O arquivo contém ransomware que criptografa todos os dados do computador e exige um pagamento para liberar os arquivos.

O objetivo é mostrar os riscos de baixar arquivos de fontes não confiáveis e a importância de backups regulares.

Como o ransomware funciona:

  • O usuário é induzido a baixar um arquivo malicioso.
  • Ao abrir o arquivo, o ransomware se instala e começa a criptografar os dados do computador.
  • Um aviso aparece exigindo pagamento (normalmente em criptomoedas) para liberar os arquivos.
  • Mesmo após o pagamento, não há garantia de que os dados serão recuperados.

Como se proteger:

  • Evite baixar arquivos de sites não confiáveis.
  • Mantenha cópias de segurança (backups) atualizadas e armazenadas fora do computador.
  • Use antivírus atualizado e mantenha o sistema operacional protegido.
  • Não clique em links desconhecidos ou suspeitos.
  • Eduque os usuários sobre os riscos e as boas práticas de segurança digital.

Soluções:

  • Fazer backup periódico dos arquivos em nuvem ou HD externo.
  • Bloquear a execução automática de arquivos desconhecidos.
  • Instalar e manter um antivírus e firewall atualizados.
  • Treinar os usuários para reconhecer riscos e evitar downloads perigosos.

Senhas Fracas

Descrição:

Um sistema tem várias contas de usuário com senhas simples como "123456" e "password". Um atacante usa um ataque de força bruta para acessar essas contas.

Como senhas fracas são exploradas:

  • Atacantes usam listas comuns de senhas para tentar adivinhar acessos.
  • Utilizam ferramentas automatizadas que testam milhares de combinações rapidamente.
  • Senhas simples ou repetidas são quebradas em segundos.

Como se proteger:

  • Utilize senhas fortes com letras, números e símbolos.
  • Evite reutilizar senhas em vários sites.
  • Implemente políticas de senha forte nos sistemas.
  • Habilite autenticação em dois fatores (2FA).
  • Utilize um gerenciador de senhas para armazená-las com segurança.

Software Desatualizado

Descrição:

Um servidor está executando uma versão antiga de um software que possui vulnerabilidades conhecidas. Um atacante explora essas falhas para obter acesso ao servidor.

Como funciona:

  • Softwares desatualizados contêm brechas de segurança já documentadas.
  • Atacantes exploram essas brechas para instalar malwares ou obter controle do sistema.

Como se proteger:

  • Mantenha todos os softwares e sistemas operacionais atualizados.
  • Habilite atualizações automáticas sempre que possível.
  • Monitore vulnerabilidades conhecidas em softwares utilizados.

Soluções:

  • Criar rotina de atualização de sistemas e programas.
  • Utilizar ferramentas de inventário de software e gestão de patches.
  • Descontinuar o uso de softwares obsoletos.

Engenharia Social

Cenário de Pretexting:

Descrição:

Um atacante liga para um funcionário fingindo ser do departamento de TI e solicita informações de login para "resolver um problema urgente". O funcionário, acreditando na urgência, fornece as informações. Demonstrar como a engenharia social pode manipular pessoas e como verificar a autenticidade das solicitações.

Como funciona:

  • O atacante cria uma identidade falsa (pretexting).
  • Usa informações reais para ganhar a confiança da vítima.
  • Finge ser alguém de autoridade ou suporte técnico.

Como se proteger:

  • Verifique a identidade do solicitante antes de fornecer informações.
  • Desconfie de solicitações urgentes ou incomuns.
  • Use canais oficiais para confirmar solicitações.

Soluções:

  • Treinamento regular sobre engenharia social e segurança da informação.
  • Implementar políticas de verificação de identidade.
  • Criar um canal seguro para relatar tentativas de engenharia social.

Cenário de Baiting:

Descrição:

Um atacante deixa um pendrive infectado em um local público. Um funcionário encontra o pendrive e o conecta ao seu computador, infectando o sistema com malware. Mostrar os riscos de conectar dispositivos desconhecidos e a importância de políticas de segurança.

Como funciona:

  • O atacante deixa um dispositivo USB em um local público.
  • Quando alguém conecta o dispositivo, o malware é instalado automaticamente.
  • O malware pode roubar dados, instalar backdoors ou criptografar arquivos.

Como se proteger:

  • Não conecte dispositivos desconhecidos ao computador.
  • Desative a execução automática de dispositivos USB.
  • Use software antivírus para escanear dispositivos antes de usá-los.
  • Eduque os usuários sobre os riscos de dispositivos desconhecidos.

Soluções:

  • Implementar políticas de segurança para dispositivos USB.
  • Utilizar software de controle de dispositivos para bloquear conexões não autorizadas.
  • Realizar treinamentos sobre segurança da informação e riscos associados a dispositivos desconhecidos.

Cenário de Violação de Dados:

Descrição:

Uma empresa descobre que dados confidenciais de clientes foram acessados por um atacante. A equipe de resposta deve identificar a origem da violação, conter o ataque, erradicar a ameaça e recuperar os dados. Como responder a uma violação de dados e a importância de um plano de resposta a incidentes.

Como funciona:

  • O atacante explora vulnerabilidades no sistema para acessar dados.
  • Rouba informações sensíveis, como senhas, dados bancários ou pessoais.
  • Pode usar os dados para extorsão, venda ou outros crimes.

Como se proteger:

  • Implemente controles de segurança adequados (firewalls, criptografia).
  • Realize auditorias e testes de penetração regularmente.
  • Eduque os funcionários sobre segurança da informação.
  • Tenha um plano de resposta a incidentes bem definido.

Soluções:

  • Implementar políticas de segurança da informação.
  • Realizar treinamentos regulares sobre segurança cibernética.
  • Monitorar e auditar sistemas regularmente.
